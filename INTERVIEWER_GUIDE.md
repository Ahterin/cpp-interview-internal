# Interviewer Guide

This document is for interviewers only.

## Goal
Evaluate practical engineering thinking in a small C++ repository.

## What this exercise is meant to test
- orientation in an existing repository
- ability to read and use documentation
- basic C++ code review skills
- ability to identify missing test scenarios
- use of fake/stub dependencies in unit tests
- responsible use of AI/search tools with clear understanding of submitted changes
- practical Git workflow awareness
- communication and reasoning during problem solving

## Expected technical findings
- README says VIP discount should be 20%
- candidate branch implements VIP discount as 25%
- VIP scenario is missing from tests
- missing negative-price decision or test
- API behavior for invalid input is not very explicit
